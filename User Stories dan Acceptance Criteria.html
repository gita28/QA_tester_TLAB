User Stories dan Acceptance Criteria

EPIC 1: Sub Sistem Perangkat IoT

User Story 1: Deteksi Suhu Lingkungan

As a system user,

I want the IoT system to detect the ambient temperature

so that I can monitor temperature changes in the environment.

Acceptance Criteria:

  • The system detects and records ambient temperature in real-time.
  • The temperature data is updated at regular intervals (e.g., every 10 seconds).
  • The temperature sensor works within the specified range (e.g., -20°C to 50°C).
  • If the sensor is malfunctioning, the system displays an error notification.

User Story 2: Pengisian Daya dari Solar Panel

As an operator,

I want the IoT device to charge its battery using a solar panel

so that the device remains operational without needing external power.

Acceptance Criteria:

  • The system shows real-time battery status.
  • The device charges successfully when exposed to sunlight.
  • The device continues functioning while charging.
  • If solar charging fails, an alert is generated.

EPIC 2: Sub Sistem Komunikasi

User Story 3: Pengiriman Data Suhu ke Back End

As a backend user,

I want the IoT system to send temperature data via 4G

so that the data can be stored and analyzed on the backend.

Acceptance Criteria:

  • The system sends real-time temperature data to the backend server.
  • Data transmission should be completed successfully via a 4G modem.
  • If transmission fails, the system retries three times before alerting an error.
  • The system uses MQTT or HTTP protocol for data transmission.

User Story 4: Penerimaan Perintah Aktivasi Sensor

As a system administrator,

I want the backend to send activation commands to the IoT device

so that sensors can be activated remotely.

Acceptance Criteria:

  • The backend system sends activation commands to the IoT device.
  • The IoT device successfully activates the sensor upon receiving the command.
  • If the command fails, the system retries and generates an error after 3 failures.
  • The communication protocol used is either MQTT or HTTP.

EPIC 3: Sub Sistem Back End

User Story 5: Menyimpan Data Sensor

As a backend administrator,

I want to store sensor data from the IoT device

so that I can use it for reporting and analysis.

Acceptance Criteria:

  • The backend system receives data from all sensors (temperature, humidity, light, images) in real-time.
  • The data is correctly stored in the Postgres database.
  • The system ensures data integrity and prevents duplication.
  • If any data transmission is incomplete, the system alerts the admin.

User Story 6: Menyimpan Gambar ke Object Storage

As a system administrator,

I want images from the IoT device to be stored in an object storage

so that the images can be accessed and processed later.

Acceptance Criteria:

  • The system receives image data from the IoT device.
  • The system stores images in the object storage service.
  • The system ensures images are stored in the correct folder or bucket.
  • If image storage fails, the system retries three times and notifies the administrator.

EPIC 4: Sub Sistem Front End

User Story 7: Menampilkan Data Sensor di Front End

As a user,

I want the Front End to display sensor data from the IoT device

so that I can monitor environmental conditions in real-time.

Acceptance Criteria:

  • The frontend system shows temperature, humidity, and light intensity data in real-time.
  • The data is displayed in a user-friendly format (e.g., graphs, tables).
  • Users can filter the data based on time (daily, hourly, weekly).
  • If new data arrives, the frontend system updates the display automatically.

User Story 8: Menampilkan Gambar dari Object Storage

As a system user,

I want to view images captured by the IoT device in the frontend

so that I can assess the environment and conditions visually.

Acceptance Criteria:

  • The frontend system retrieves images from the object storage.
  • The images are displayed in a gallery or slideshow format.
  • Users can zoom in and out of the images for better detail.
  • If the image cannot be retrieved, an error message is displayed.
